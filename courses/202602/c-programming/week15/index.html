    15주차 · C ↔ ROS2 · 로봇 · 기말 프로젝트¶

    C언어 · 미래모빌리티학과 | CLO2·CLO4·CLO5

    학습 목표¶

    • 배운 C(구조체·포인터·배열)가 로봇 통신의 핵심 코드임을 체험한다.
    • C 프로그램과 ROS2 노드가 데이터를 주고받게 한다.
    • (캡스톤) Arduino + C + Stella N2 로봇으로 자율 동작을 구현·발표한다.

    강의 해설¶

    15주차는 한 학기 동안 배운 문법이 실제 로봇 시스템에서 어떻게 만나는지 확인하는 종합 주차다. 처음에는 printf로 숫자를 출력했지만, 이제는 센서 배열을 분석하고 구조체로 판단 결과를 묶고 ROS2 토픽으로 주행 명령을 내보낸다. 이때 새로 등장하는 ROS2가 주인공처럼 보일 수 있지만, 실제 판단의 중심은 여전히 C로 작성한 함수다.

    ROS2의 토픽은 프로그램들이 데이터를 주고받는 통로다. /scan은 LiDAR 거리 배열을 보내고, /cmd_vel은 로봇의 선속도와 각속도를 보낸다. 여기서 /scan의 ranges[]는 10주차 배열, 가장 가까운 장애물 탐색은 6주차 반복문, 정지/회피 판단은 5주차 조건문, 판단 결과 묶음은 14주차 구조체, 배열 전달은 12~13주차 포인터와 연결된다.

    기말 프로젝트의 목표는 멋진 기능을 많이 붙이는 것이 아니라, 작은 기능을 안전하고 설명 가능하게 완성하는 것이다. 로봇은 실제로 움직이는 장비이므로 코드 품질과 안전 절차가 기능만큼 중요하다. 바퀴를 띄운 상태에서 /cmd_vel 값을 먼저 확인하고, 속도는 작게, 정지거리는 크게 시작한다. 발표에서는 "어떤 C 함수가 어떤 판단을 했고, 그 결과가 어떤 ROS2 메시지로 바뀌었는가"를 설명할 수 있어야 한다.

    3시간 강의 운영 포인트¶

    • 0~25분: 1~14주차 개념을 ROS2 로봇 흐름으로 다시 묶는다. 배열, 포인터, 구조체, 함수가 각각 어디에 쓰이는지 한 장 그림으로 정리한다.
    • 25~80분: ROS2 토픽, /scan, /cmd_vel, 시리얼 브리지, micro-ROS 선택지를 설명한다. 새 문법을 늘리기보다 C 함수가 판단의 중심임을 계속 강조한다.
    • 80~135분: LiDAR 배열 분석 또는 C↔ROS2 브리지 실습을 진행한다. 먼저 PC에서 C 함수 단위 테스트를 통과시키고, 그다음 로봇 또는 시뮬레이터에 연결한다.
    • 135~180분: 기말 프로젝트 발표와 안전 점검을 진행한다. 속도 제한, 정지 조건, 장애물 거리 기준, 실패 시 수동 정지 절차를 발표 항목에 포함한다.

    강의 본문 보강¶

    개념을 더 풀어 설명하기¶

    마지막 주차의 핵심은 ROS2라는 새 기술을 많이 외우는 것이 아니다. 한 학기 동안 배운 C 문법이 실제 로봇 소프트웨어 안에서 다시 등장한다는 사실을 확인하는 것이다. /scan의 ranges[]는 배열이고, 가장 가까운 장애물을 찾는 과정은 반복문이며, 정지와 회피 판단은 조건문이다. 판단 결과를 묶으면 구조체가 되고, 큰 배열을 함수에 넘길 때는 포인터가 쓰인다.

    ROS2 토픽은 프로그램 사이의 데이터 통로다. LiDAR 노드는 /scan으로 거리 배열을 보내고, 주행 제어 노드는 /cmd_vel로 속도 명령을 보낸다. 이 수업에서는 ROS2 전체를 깊게 배우기보다, C로 작성한 판단 함수가 ROS2 메시지와 어떻게 연결되는지 보는 데 집중한다.

    실습 흐름¶

    먼저 PC에서 scan_logic.c만 따로 읽고 테스트한다. 이 파일은 ROS2를 몰라도 이해할 수 있는 순수 C 로직이어야 한다. 그 다음 stella_n2_bridge.cpp를 보며 C 함수의 결과가 /cmd_vel 메시지로 바뀌는 과정을 확인한다. 마지막으로 테스트용 /scan 데이터를 발행하고 /cmd_vel 출력이 어떻게 바뀌는지 관찰한다.

    학생 활동¶

    • ranges[] 배열에서 가장 가까운 거리와 위치를 찾는 코드를 손으로 추적한다.
    • 정지거리 기준을 바꾸었을 때 주행 명령이 어떻게 달라지는지 비교한다.
    • /scan 테스트 메시지를 3가지 상황으로 만들어 본다. 안전, 주의, 정지 상황을 각각 포함한다.
    • 발표 자료에는 "어떤 C 함수가 어떤 판단을 했는가"를 반드시 넣는다.

    안전과 평가 포인트¶

    • 실제 로봇 연결 전에는 반드시 테스트 토픽과 시뮬레이션 흐름으로 확인한다.
    • 첫 실행은 바퀴를 띄운 상태에서 진행한다.
    • 속도는 작게, 정지거리는 크게 시작한다.
    • 기능이 많아도 설명할 수 없는 코드는 좋은 프로젝트가 아니다. C 코드 품질, 안전성, 재현 가능한 실행 절차가 함께 평가된다.

    1. 이론¶

    1.1 ROS2와 C의 관계¶

    • ROS2는 로봇 SW의 표준 미들웨어. 노드들이 토픽(topic) 으로 데이터를 주고받는다(발행 publish / 구독 subscribe).
    • C 진입점: rcl(ROS2의 C 코어), rclc(MCU용 순수 C, micro-ROS). → 우리가 배운 C가 그대로 쓰인다.

    1.2 두 가지 연동 방식¶

    보드는 시리얼 송수신, PC의 C/C++ 브리지가 ROS2로 변환. 설치 간단, C 학습(파싱·구조체) 집중. 핵심 C 함수: parse_sensor_line()(구조체로 파싱), analyze_scan()(배열 분석).

    보드가 직접 ROS2 노드. rclc=순수 C. UNO R4 공식 지원(RAM 32KB라 경량).

    1.3 LiDAR 배열 → C로 판단 (자율주행의 심장)¶

    Stella N2의 LiDAR는 거리 배열을 /scan으로 보낸다. 이를 C로 분석해 주행을 정한다. 

    ScanResult analyze_scan(const float *ranges, int n, float stop_dist) {
    // 1) 배열 순회로 최솟값(가장 가까운 거리) 탐색
    // 2) 거리·방향 → 전진(F)/좌회피(L)/우회피(R)/정지(S)
}

    배운 것 총동원: 배열 순회 + 조건 판단 + 구조체 반환 + 포인터 인자. 상세: C↔ROS2 & Stella N2 로봇

    2. 핵심 용어 정리¶

    용어 설명
    ROS2 로봇 SW 표준 미들웨어
    노드/토픽 실행 단위 / 데이터 통로
    publish/subscribe 발행 / 구독
    /scan, /cmd_vel LiDAR 거리 / 주행 속도 토픽
    브리지 시리얼↔ROS2를 잇는 프로그램
    micro-ROS / rclc MCU용 ROS2 / 순수 C API

    실습 15-2 · C↔ROS2 연동¶

    시리얼 브리지로 보드 ↔ ROS2 토픽 pub/sub 확인.

    기본 실행 흐름:

    source /opt/ros/jazzy/setup.bash
mkdir -p ~/cprog_ws/src
cp -r courses-src/c-programming-202602/docs/code/ros2/stella_n2_bridge ~/cprog_ws/src/
cd ~/cprog_ws
colcon build --packages-select stella_n2_bridge
source install/setup.bash
ros2 run stella_n2_bridge stella_n2_bridge

    테스트용 /scan:

    ros2 topic pub /scan sensor_msgs/msg/LaserScan "{ranges: [1.2, 0.9, 0.4, 0.8, 1.1], range_min: 0.05, range_max: 8.0}" -r 2
ros2 topic echo /cmd_vel

    먼저 scan_logic.c를 읽는다. 이 파일은 ROS2를 몰라도 이해할 수 있는 순수 C 코드다. 그 다음 stella_n2_bridge.cpp를 보면 C 함수의 결과가 /cmd_vel 메시지로 바뀌는 과정을 볼 수 있다.
