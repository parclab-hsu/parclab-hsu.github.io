    운영과 Q&A¶

    Q&A와 적용 클리닉¶

    단순한 자유 질문 시간이 아니라, 각자의 업무 자료로 직접 옮겨가는 시간입니다.

    세 가지 범주(내 자료 진단 / 분석 질문 만들기 / 발표자료 바꾸기)로 구조를 잡아 훨씬 차분하게 마무리할 수 있습니다.

    마지막 1시간은 이렇게 진행됩니다¶

    마지막 1시간은 아래 세 가지 클리닉 유형 중 하나를 선택해 진행합니다.

    • A. 내 자료 진단 — 파일 구조가 복잡한 엑셀 / 학과·수업 운영 데이터 / 설문 응답 데이터
    • B. 분석 질문 만들기 — 무엇을 비교할지 / 어떤 기준으로 나눌지 / 어떤 검증이 필요한지
    • C. 발표자료 바꾸기 — 결론형 제목 / 차트 선택 / HTML 배포 흐름

    클리닉 진행 방식은 다음과 같습니다.

    • 참여 방식 — 질문은 A/B/C 중 하나 / 한 화면 5분 / 공통 프롬프트 남기기
    • 학습 포인트 — 답보다 질문 구조 / 좋은 사례는 함께 저장

    5분 클리닉은 이런 순서로 도움을 받습니다¶

    한 사람의 문제가 전체 학습으로 이어지는 구조입니다.

    5분 클리닉은 아래 순서로 진행되어 개별 케이스가 전체 학습으로 이어집니다.

    시간 단계
    1분 문제 상황 듣기
    1분 파일·화면 상태 확인
    2분 프롬프트 함께 수정
    1분 전체 적용 원칙 정리
    • 좋은 클리닉 — 내 문제를 해결하면서도 전체가 가져갈 수 있는 질문 문장이 남습니다.
    • 참고 — 한 사람의 PC 문제를 오래 붙잡으면 전체 흐름이 멈출 수 있으니, 빠르게 다음 단계로 넘어갑니다.

    예상 질문과 답변¶

    자주 묻는 질문과 핵심 답변을 미리 확인해 두세요.

    질문 짧은 답변
    코딩을 몰라도 가능한가요? 가능합니다. 다만 결과를 검증하는 질문을 배워야 합니다.
    개인 성적표를 넣어도 되나요? 식별정보와 민감정보는 익명화 전에는 넣지 않는 것이 원칙입니다.
    AI가 틀리면 어떻게 하나요? 행 수, 합계, 샘플 행, 기준연도를 확인하게 하고 근거를 요구합니다.
    HTML이 PPT보다 어려운가요? 생성은 AI에게 맡기고, 구조와 검증 기준을 분명히 제시하면 됩니다.
    GitHub Pages가 어려우면요? 수업 중에는 로컬 HTML로 확인하고, 배포는 사후 과제로 처리해도 됩니다.
    엑셀이 안 읽히면요? 일반 판독 실패인지, 내부 구조 문제인지 먼저 진단하게 합니다.

    질문은 프롬프트로 바꿔서 남깁니다¶

    워크숍에서 나온 질문은 바로 재사용 가능한 프롬프트로 전환됩니다. 아래 예시를 참고해 직접 활용해 보세요.

    참가자 질문 프롬프트로 바꾼 문장
    엑셀이 이상하게 읽혀요 이 파일을 읽을 때 실제 데이터 범위와 헤더 행을 진단하고, 왜 일부만 읽혔는지 설명해줘.
    이 숫자가 맞는지 모르겠어요 원본 합계 행과 계산 결과를 비교하고, 차이가 있으면 원인을 후보별로 정리해줘.
    차트를 뭘로 해야 할까요? 이 메시지를 보여주기에 적합한 차트 후보 3개와 각각의 장단점을 제안해줘.
    발표 제목이 밋밋해요 현재 제목을 결론형 제목으로 바꾸고, 데이터가 말하지 않는 원인은 단정하지 않게 다듬어줘.

    문제가 생겼을 때의 복구 동선¶

    자주 생기는 상황을 미리 정리했습니다. 당황하지 말고 아래 순서대로 확인해 보세요.

    • 설치 실패 — 짝꿍 화면으로 관찰 / 개인 설치는 마지막
    • 파일 못 찾음 — 작업 폴더 확인 / data 위치와 파일명 확인
    • 분석 결과 이상 — 행 수·합계 확인 / 원본 샘플·기준연도 확인
    • HTML 생성 지연 — 구성안 먼저 만들기 / 슬라이드 1-2장만 제작
    • GitHub Pages 지연 — 로컬 HTML로 발표 / 배포는 사후 과제
    • 네트워크 제한 — 준비한 분석 결과 사용 / 프롬프트 설계 중심 진행

    마무리 메시지¶

    오늘 워크숍 전체를 관통하는 핵심 메시지입니다.

    AI 에이전트 활용 능력은 자료를 확인시키고, 이상한 점을 찾게 하고, 검증해 결과물로 연결하는 능력입니다.

    이 능력은 다음 다섯 단계로 구성됩니다.

    단계 핵심 질문
    확인 무엇이 있나
    질문 무엇을 알고 싶나
    분석 어떻게 계산했나
    검증 맞는가
    전달 어떻게 말할까
