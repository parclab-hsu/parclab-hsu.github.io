    워크숍 소개¶

    오늘 끝나면 남길 결과물¶

    완성도보다 흐름을 몸으로 익히는 것이 목표입니다. 오늘 워크숍이 끝나면 세 가지 결과물이 손에 남습니다.

    1. 데이터 지도 — data 폴더의 엑셀 파일이 어떤 주제와 연도를 담고 있는지 표로 정리합니다. 완성 형태: 파일명, 연도, 시트, 행/열 수
    2. 검증된 인사이트 — AI가 만든 분석 결과를 행 수, 합계, 기준연도, 샘플 행으로 다시 확인합니다. 완성 형태: 근거가 붙은 결론 문장 1개
    3. 웹 슬라이드 초안 — 분석 결과를 발표 흐름으로 바꾸고 GitHub Pages로 공유할 수 있는 형태까지 연결합니다. 완성 형태: HTML 파일과 실제 공유 URL

    오늘의 성공 기준 — 완벽한 산출물보다, AI에게 일을 맡기고 결과를 의심하고 다시 고치는 절차를 익히는 것입니다.

    진행 방식: 짧게 맡기고 바로 확인¶

    긴 설명보다 작은 작업 단위를 반복합니다. 워크숍 전체를 아우르는 네 단계 루프를 짧게 반복하면서 진행합니다.

    1. 보기 — 3-5분 시연
    2. 따라 하기 — 같은 프롬프트
    3. 확인 — 행 수·합계 점검
    4. 고치기 — 추가 지시

    오늘 마음가짐으로 기억해 두세요.

    • 오늘 마음가짐 — 처음부터 완성하려 하지 않음 / 첫 결과를 보고 다음 질문 만들기 / 결과보다 확인 과정을 말로 남기기

    오늘 4시간은 이렇게 씁니다¶

    설치가 끝나면 바로 실제 자료를 다룹니다. 4시간을 네 블록으로 나눠 진행합니다.

    • 1시간 — 설치와 작업 폴더 열기
    • 1시간 — 데이터 분석
    • 1시간 — 웹 발표자료 제작
    • 1시간 — Q&A 클리닉

    각 블록을 어떻게 활용할지 미리 알아 두면 좋습니다.

    • 앞 1시간은 설치가 늦어지더라도 당황하지 않아도 됩니다. 여유 있게 설정을 마칩니다.
    • 중간 2시간은 데모-실습-검증을 짧게 반복합니다.
    • 마지막 1시간은 각자의 업무 자료로 옮겨갈 질문을 정리합니다.

    전체 시간표¶

    설치 편차를 감안해 앞 1시간은 여유 있게, 나머지는 데모-실습-검증으로 짧게 반복합니다.

    시간 세션 진행 내용 참가자 결과
    00:00-00:10 문제 설정 AI 에이전트 작업 흐름 소개 오늘 다룰 자료 이해
    00:10-00:50 GitHub/Codex 준비 GitHub 가입, Codex 접속, 권한 설명 계정 로그인과 작업 폴더 연결
    00:50-01:00 첫 프롬프트 파일 목록 파악 시연 데이터 목록 요약
    01:00-02:00 데이터 분석 정제, 차트, 검증 프롬프트 핵심 인사이트 1개
    02:00-03:00 웹 발표자료 제작 분석 결과를 웹 슬라이드로 전환 웹 발표자료 초안
    03:00-04:00 Q&A 클리닉 개인/학과 자료 적용 상담 다음 실천 과제

    미리 준비하면 좋은 것¶

    설치에 시간을 빼앗기지 않도록, 워크숍 전에 아래만 확인해 두면 좋습니다.

    • 계정 — GitHub 가입용 이메일과 OpenAI 계정 로그인을 미리 확인합니다.
    • 자료 — 실습용 엑셀(data 폴더)을 여기서 한 번에 내려받으세요. 압축을 풀면 data 폴더가 나옵니다. 파일명의 한글·공백은 그대로 둡니다.
    • 네트워크 — GitHub, OpenAI, Pages 접속이 기관 네트워크에서 막히지 않는지 확인합니다.
