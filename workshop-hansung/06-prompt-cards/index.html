    프롬프트 카드¶

    복사해서 바로 쓸 수 있는 실습 문장들을 모아 두었습니다. 워크숍 중에 활용하거나, 나중에 실제 업무에서 꺼내 쓸 수 있도록 정리한 부록입니다.

    프롬프트 카드 1: 탐색¶

    파일을 읽기 전후에 근거를 남기게 합니다. 아래 두 프롬프트는 데이터 탐색 단계에서 가장 먼저 사용하는 문장들입니다. 첫 번째는 파일 구조를 체계적으로 파악할 때, 두 번째는 파일이 예상대로 읽히지 않을 때 원인을 진단하는 데 활용합니다.

    데이터 탐색

    data 폴더의 모든 xlsx 파일을 열어보고,
1) 파일명, 2) 주제, 3) 파일명 연도, 4) 실제 데이터 행 수,
5) 헤더가 시작되는 행, 6) 분석 시 주의점을 표로 정리해줘.

결과를 내기 전에 파일을 실제로 읽었는지 근거를 보여줘.

    오류 진단

    엑셀 파일을 읽었을 때 한 칸만 보이거나 행 수가 이상하면,
파일 내부 구조와 실제 셀 태그 수를 확인해서 원인을 설명해줘.
가능하면 데이터를 복원하는 방법도 제안해줘.

    프롬프트 카드 2: 분석과 검증¶

    분석 질문과 검증 질문을 붙여 씁니다. 분석 프롬프트로 결과를 얻은 뒤, 검증 프롬프트를 이어서 실행하는 습관을 들이는 것이 이 단계의 핵심입니다.

    분석

    외국학생 현황 2023-2025 파일을 분석해서
1) 총원 추이, 2) 국가별 상위 5개, 3) 증가분 상위 국가,
4) 해석상 주의점을 정리해줘.

표와 차트에 쓸 수 있는 데이터 형태로 결과를 만들어줘.

    검증

    방금 만든 분석 결과를 검증해줘.
원본 합계 행과 계산 결과를 비교하고,
기준연도/파일명 연도/분모/단위에서 해석상 주의할 점을 적어줘.
검증이 불충분한 부분은 '추가 확인 필요'라고 표시해줘.

    프롬프트 카드 3: 웹 발표자료¶

    분석 결과를 GitHub Pages에 올릴 결과물로 바꿉니다. 먼저 구성안 프롬프트로 슬라이드 뼈대를 잡은 뒤, HTML 생성 프롬프트로 실제 파일을 만듭니다.

    구성안

    지금까지 분석한 내용을 가지고
'AI 에이전트로 대학 행정 데이터를 분석하고 발표자료를 만드는 흐름'이라는 주제의
8장 발표자료 구성을 만들어줘.

슬라이드마다
1) 제목, 2) 핵심 메시지 한 문장, 3) 들어갈 차트나 표,
4) 발표할 때 말할 진행 멘트를 적어줘.
청중은 대학 교수이고, 실습형 워크숍에 맞는 자연스러운 말투로 작성해줘.

    HTML 생성

    위 구성안으로 GitHub Pages에 올릴 수 있는 웹 발표자료를 만들어줘.
조건은 다음과 같아.

- 발표자료/slide/index.html, styles.css, app.js로 구성
- 16:9 발표 화면
- 키보드로 이전/다음 이동
- 발표자 메모 토글
- 외국학생 총원 추이, 국가별 상위 국가, 중도탈락률, 성적분포 차트를 SVG로 렌더링
- 마지막에는 참가자가 다시 써볼 수 있는 프롬프트 카드 섹션 포함
- 외부 라이브러리 없이 GitHub Pages에서 바로 열리게 작성

    개인정보와 민감정보 안내 문구¶

    워크숍 시작 전에 짧게 읽어도 좋습니다. 실습 이후 실제 업무 데이터에 AI 도구를 적용할 때 지켜야 할 기준을 미리 확인해 두세요.

    오늘 실습 데이터

    공개 가능한 예제 데이터만 사용합니다. 이름, 학번, 연락처, 개별 성적, 상담 내용, 민감 설문 응답은 AI 도구에 입력하지 않습니다.

    업무 자료를 다룰 때

    기관 정책과 데이터 등급을 먼저 확인합니다. 필요 최소한의 범위에서 집계·익명화한 뒤 분석합니다.

    핵심 — 개인정보는 '쓰지 말자'보다, 집계·익명화해 쓸 수 있는 상태로 만드는 것이 핵심입니다.
