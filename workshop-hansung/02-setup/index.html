    환경 준비¶

    GitHub는 오늘의 배포 공간입니다¶

    GitHub를 처음 쓴다면 저장소보다 '링크로 나누는 작업 폴더'로 이해하면 편합니다.

    • 파일 보관 — HTML, CSS, JS 파일을 웹에 올릴 수 있는 공간입니다. 오늘 만든 발표자료 파일이 여기에 놓입니다. 참가자는 링크 하나로 결과물을 다시 엽니다.
    • 변경 기록 — 수정할 때마다 어떤 파일이 바뀌었는지 기록됩니다. AI가 만든 결과를 되돌아볼 때 유용합니다. 실수했을 때 이전 상태를 확인할 수 있습니다.
    • Pages 배포 — 정적 HTML 파일을 별도 서버 없이 웹 주소로 공개할 수 있습니다. 수강생은 링크 하나로 결과물을 확인합니다. 오늘은 GitHub를 배포 공간으로만 이해해도 충분합니다.

    짧은 설명 — GitHub는 개발자만 쓰는 공간이 아니라, 오늘은 웹 발표자료를 올리고 공유하는 장소로 사용합니다.

    GitHub 계정부터 준비합니다¶

    아래 순서대로 가입과 이메일 인증까지 진행합니다.

    • 가입 순서 — 접속: github.com/signup 입력: 계정 정보와 Country/Region 생성: Create account 후 이메일 인증
    • 현장 체크 — 사용자명은 공개 주소에 포함 무료 계정으로 충분 메일이 없으면 스팸함 확인

    가입 후 확인할 것¶

    가입 완료와 실습 가능 상태는 다릅니다.

    • 로그인 — 우측 상단 프로필 확인 안 보이면 로그인부터
    • 이메일 인증 — 인증 메일 클릭 전에는 저장소·배포 제한 가능
    • 사용자명 — 공개 주소에 포함 실명·학번 그대로 사용 금지
    • 결제 화면 — 무료 계정으로 충분 유료 안내는 건너뛰기

    막힐 때 — 인증이 늦으면 개인 메일을 써 보세요.

    Codex에서 오늘 할 일은 네 가지입니다¶

    기능 목록보다 작업 흐름을 먼저 파악합니다.

    단계 작업
    1. 폴더 열기 작업 폴더 연결
    2. 자료 읽기 data 엑셀 확인
    3. 파일 만들기 slide 파일 생성
    4. 결과 확인 Browser/Diff 검토
    • 기억할 것 — Codex는 채팅창이 아니라 작업 폴더 안에서 파일을 읽고 고치고 확인하는 에이전트입니다. 오늘은 data 폴더와 slide 폴더를 오가며 일하게 합니다.

    Codex 설치와 접속에서 확인할 것¶

    제품 메뉴는 바뀔 수 있으니, 오늘 필요한 개념 위주로 확인합니다.

    • 계정 — OpenAI 계정으로 로그인되어 있는지 확인합니다. 계정별 기능 차이는 본인 화면을 기준으로 확인하세요.
    • 작업 폴더 — Codex에서 이 저장소 폴더를 열었는지 확인합니다. 다른 폴더를 열면 data 파일을 찾지 못합니다.
    • 네트워크 — GitHub, OpenAI, Pages 접속이 기관 네트워크에서 막히지 않는지 확인합니다.
    • 권한 요청 — 파일 쓰기, 명령 실행, 네트워크 접근 요청이 뜨면 이유와 범위를 읽고 승인합니다.

    Codex Desktop App 버튼을 먼저 읽습니다¶

    화면에서 길을 잃지 않도록 프로젝트, 스레드, 확인 도구, 승인 요청 위치를 먼저 파악합니다.

    • 처음 볼 곳 — 프로젝트: 오늘 작업 폴더 새 스레드: 새 작업 시작 Mode: Local/Worktree 선택
    • 작업 중 볼 곳 — Browser/Terminal: 화면·명령 확인 Diff/Approval: 변경·권한 확인
    • 흐름 요약 — 폴더 열기 → 일 맡기기 → 결과 확인, 권한은 필요한 것만 허용합니다.

    Codex 화면과 버튼 한눈에 보기¶

    실제 Codex 데스크톱 앱 화면으로 자주 쓰는 버튼만 짚어 봅니다. 앱은 업데이트될 수 있으니 버튼 이름보다 역할 위주로 봐 주세요.

    메인 화면과 입력창¶

    가운데 입력창에 할 일을 자연어로 적고, 아래 버튼으로 작업 방식을 정합니다.

    • 입력창("무엇이든 해보세요") — 시킬 일을 한국어로 그대로 적습니다.
    • + — 파일·이미지 등 참고할 자료(컨텍스트)를 추가합니다.
    • 나 대신 승인 — 권한(승인) 모드를 정합니다. 어디까지 자동으로 허용할지 고르는 곳입니다.
    • 5.5 매우 높음 — 사용할 모델과 추론(생각의 깊이) 강도를 고릅니다.
    • 🎤 음성 / ↑ 전송 — 음성으로 입력하거나, 적은 내용을 전송해 작업을 시작합니다.
    • 프로젝트에서 작업하기 — 작업할 폴더(프로젝트)를 연결합니다. 오늘 작업 폴더가 맞는지 여기서 확인합니다.

    왼쪽 내비게이션¶

    • 새 채팅 — 새 작업(스레드)을 시작합니다. 주제가 바뀌면 새로 엽니다.
    • 검색 — 지난 스레드와 내용을 찾습니다.
    • 플러그인 — 외부 앱·도구를 연결합니다.
    • 자동화 — 반복 작업을 예약하거나 자동으로 실행합니다.
    • 설정(맨 아래) — 계정과 환경을 설정합니다.

    승인·권한 모드¶

    '나 대신 승인'을 누르면 권한 범위를 고를 수 있습니다. AI 에이전트가 파일과 명령을 직접 다루므로, 워크숍에서 가장 중요하게 짚는 화면입니다.

    • 승인 요청 — 외부 파일 수정·인터넷 사용 시 항상 확인합니다. 가장 보수적입니다.
    • 나 대신 승인 — 위험하다고 감지된 작업에만 승인을 요청합니다. 오늘 실습의 기본값으로 적당합니다.
    • 전체 권한 — 인터넷과 모든 파일에 제한 없이 접근합니다. 편하지만 위험하니 워크숍에서는 권하지 않습니다.
    • 사용자 지정(config.toml) — 설정 파일로 권한을 세밀하게 정합니다.

    워크숍 기준 — '승인 요청' 또는 '나 대신 승인'으로 시작하고, 작업 폴더 밖 접근과 민감정보 접근은 허용하지 않습니다.

    모델·추론 강도¶

    • 추론(낮음·중간·높음·매우 높음) — 더 깊이 생각할수록 정확해지지만 느려집니다. 어려운 분석은 높게, 간단한 작업은 낮게 둡니다.
    • 모델(GPT-5.5) — 사용할 모델을 고릅니다.
    • 속도 — 응답 속도를 우선하는 옵션입니다.

    새 스레드는 작업 단위로 나눕니다¶

    한 스레드에 모든 일을 넣기보다, 목표가 바뀔 때 새로 시작합니다.

    • 설치 확인 스레드 — 폴더가 열렸는지, data 폴더가 보이는지, 파일 목록을 읽을 수 있는지 확인합니다. 완료 기준: 파일 목록 요약이 나온다.
    • 데이터 분석 스레드 — 엑셀 구조 파악, 정제, 차트 데이터 생성, 검증 질문을 진행합니다. 완료 기준: 숫자와 근거가 같이 나온다.
    • 발표자료 제작 스레드 — 구성안, HTML/CSS/JS 생성, 화면 검수, GitHub Pages 배포 확인을 맡깁니다. 완료 기준: 브라우저에서 열린다.

    비유 — 스레드는 노트북의 새 페이지처럼 생각하면 됩니다. 주제가 바뀌면 새 페이지를 열어 맥락을 정리합니다.

    권한과 보안: 처음에 함께 정할 기준¶

    AI 에이전트는 파일과 명령을 다룰 수 있으므로 승인 전 확인을 습관화합니다.

    단계 내용
    1. 읽기 파일 내용 파악
    2. 쓰기 새 파일 생성
    3. 명령 실행 분석/변환
    4. 승인 허용 범위 확인
    • 함께 지킬 기준 — AI가 요청하는 권한은 왜 필요한지 보고 허용합니다. 오늘은 워크숍 폴더 안의 실습 파일을 읽고 결과물을 만드는 범위에서만 진행합니다.
    • 민감정보 원칙 — 학생 개인 식별정보, 성적 원자료, 상담 내용, 민감 설문 응답은 익명화 전에는 넣지 않습니다. 공개 가능한 예제 데이터로 먼저 흐름을 익힙니다.

    승인 요청은 세 문장으로 읽습니다¶

    허용 버튼을 누르기 전에 범위와 이유를 먼저 확인합니다.

    • 무엇을 하려는가 — 파일을 읽는지, 파일을 쓰는지, 명령을 실행하는지 먼저 봅니다. 예: 엑셀 읽기, HTML 수정, 브라우저 검수
    • 어디까지 접근하는가 — 워크숍 폴더 안인지, 폴더 밖인지, 네트워크 접근이 필요한지 확인합니다. 민감정보 폴더 접근은 허용하지 않습니다.
    • 왜 필요한가 — 분석, 미리보기, 배포처럼 현재 작업과 직접 연결되는 이유가 있는지 묻습니다. 이유가 불명확하면 설명을 다시 요구합니다.

    추가로 다음 세 가지를 기억합니다.

    • 모르면 거절하는 것이 아니라, Codex에게 왜 필요한지 설명하게 합니다.
    • 워크숍 중에는 작업 폴더 밖 접근과 민감정보 접근을 허용하지 않습니다.
    • 승인 과정을 직접 보면 AI 에이전트의 작동 범위를 이해하기 좋습니다.

    첫 프롬프트: 폴더를 열었는지 확인¶

    설치가 끝났다면 곧바로 아래 프롬프트로 파일 목록을 읽게 해 봅니다.

    실습 프롬프트:

    data 폴더 안의 파일들을 확인하고,
각 파일이 어떤 주제와 연도인지 표로 정리해줘.
파일을 읽기 전에 예상되는 분석 질문도 5개 제안해줘.
    • 기대 결과 — 4개 주제 × 3개년 구조를 인식한다. 전공과목 성적 분포, 입학전형, 외국학생 현황, 중도탈락을 구분한다. 분석 전에 질문을 먼저 만든다.
    • 확인 포인트 — 파일 경로가 다르면 결과에서 바로 드러납니다. AI가 없는 파일을 말하면 실제 파일명만 근거로 하라고 다시 지시하세요.
