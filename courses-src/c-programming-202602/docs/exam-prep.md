# 📝 정보처리기사(실기) C — 코드 추적 유형 연습

> 정보처리기사 실기 C 파트는 **"이 코드의 실행 결과는?"**(출력 예측)이 핵심이다.
> 아래는 **출제 경향에 맞춰 새로 만든 원본 유형 문제**이며(실제 기출 복제 아님), 모든 코드는 `gcc`로 실행해 정답을 검증했다.
> 먼저 **손으로 추적**해 답을 적고, 접힌 해설을 펼쳐 비교하자.

---

## 출제 경향 한눈에

| 주제 | 핵심 함정 |
|------|-----------|
| 포인터·배열 | `*(p++)` 후위 증가, `p[-1]`, 배열=주소 |
| 재귀 | 종료 조건, 호출 순서 |
| 구조체 포인터 | `.` vs `->`, 멤버 갱신 순서 |
| 비트 연산 | `<<`, `&`, `^`, 16진수↔10진수 |
| 2차원 배열 | 인덱스 순회, 조건 누적 |
| 문자열 | `'\0'` 종료, 포인터 순회 |
| 전역·static | 호출 사이 값 유지, 후위 증감 |
| 형변환 | 정수 나눗셈, 캐스팅 위치 |

---

## 핵심 이론 요약 (함정 위주)

- **포인터**: 배열 이름 `a`=`&a[0]`, `*(a+i)==a[i]`. 후위 `*(p++)`는 "지금 값 쓰고 이동".
- **형변환**: `int/int`는 소수 버림(`7/2==3`). 실수는 `(double)a/b`.
- **비트**: `a<<n`=`a×2ⁿ`, `&`(둘 다 1)·`|`(하나라도)·`^`(다르면 1).
- **구조체**: `p->m`==`(*p).m`. 값은 `.`, 포인터는 `->`.
- **static/전역**: 호출 사이 값 유지. `g--`는 후위라 현재 값을 먼저 쓴다.
- **재귀**: 종료 조건부터 손으로 펼친다.

## 유형 문제 10선 (원본 · 실행 검증)

### 문제 1 · 포인터와 후위 증가
```c
int a[] = {10,20,30,40,50};
int *p = a;
printf("%d ", *(p+2));
printf("%d ", *(p++));
printf("%d ", *p);
printf("%d",  *p + 1);
```
??? success "정답: `30 10 20 21`"
    - `*(p+2)` → `a[2]` = **30** (p는 아직 안 움직임).
    - `*(p++)` → 지금 값 `a[0]`=**10** 출력 후 p가 `a[1]`로 이동.
    - `*p` → `a[1]` = **20**.
    - `*p + 1` → 20+1 = **21**.

### 문제 2 · 재귀 (3항 합)
```c
int f(int n){ if(n<3) return n; return f(n-1)+f(n-2)+f(n-3); }
// f(5) 는?
```
??? success "정답: `11`"
    - `f(0)=0, f(1)=1, f(2)=2`
    - `f(3)=2+1+0=3`, `f(4)=3+2+1=6`, `f(5)=6+3+2=`**11**

### 문제 3 · 구조체 포인터 멤버 갱신
```c
struct P { int x, y; };
struct P a = {3, 5};
struct P *p = &a;
p->x += 2;
p->y = p->x * 2;
printf("%d %d", a.x, a.y);
```
??? success "정답: `5 10`"
    - `p->x += 2` → x=**5**. `p->y = p->x*2` → **바뀐** 5×2=**10**.

### 문제 4 · 비트 연산
```c
int a = 0x0F;             // 1111 = 15
printf("%d %d %d", a<<2, a&0x09, a^0x05);
```
??? success "정답: `60 9 10`"
    - `a<<2`=15×4=**60**, `a&0x09`=`1111&1001`=**9**, `a^0x05`=`1111^0101`=`1010`=**10**.

### 문제 5 · 2차원 배열 조건 누적
```c
int a[2][3] = {{1,2,3},{4,5,6}};
int sum = 0;
for (int i=0;i<2;i++)
  for (int j=0;j<3;j++)
    if ((i+j)%2==0) sum += a[i][j];
// sum 은?
```
??? success "정답: `9`"
    - `(i+j)` 짝수 칸: (0,0)=1, (0,2)=3, (1,1)=5 → 1+3+5=**9**.

### 문제 6 · 문자열 순회
```c
char s[] = "PROCESS";
char *p = s;
int cnt = 0;
while (*p != '\0') { if (*p=='S') cnt++; p++; }
printf("%d %d", cnt, (int)(p - s));
```
??? success "정답: `2 7`"
    - `'S'` 개수 = **2**. 끝나면 p는 `'\0'` 위치 → `p-s`=길이=**7**.

### 문제 7 · 전역 + static
```c
int g = 5;
int f(void){ static int s = 1; s *= 2; return s + (g--); }
// f() 를 세 번 호출하면?
```
??? success "정답: `7 8 11`"
    - 1회: s=2, g=5사용(후 4) → **7**. 2회: s=4, g=4(후 3) → **8**. 3회: s=8, g=3(후 2) → **11**.
    - `static s`는 값 유지(2→4→8), `g--`는 후위라 현재 값 먼저 사용.

### 문제 8 · 정수 나눗셈 vs 캐스팅
```c
int a = 7, b = 2;
double r  = a / b;
double r2 = (double)a / b;
printf("%.1f %.1f", r, r2);
```
??? success "정답: `3.0 3.5`"
    - `a/b`는 정수끼리 나눠 3 → **3.0**. `(double)a/b`는 실수 나눗셈 → **3.5**.
    - 함정: 대입 대상이 double이어도 **연산 시점**의 자료형이 결과를 정한다.

### 문제 9 · 포인터 산술과 음수 인덱스
```c
int a[] = {1,2,3,4};
int *p = a + 1;
printf("%d ", p[1]);
printf("%d ", *(a+3));
printf("%d",  p[-1]);
```
??? success "정답: `3 4 1`"
    - `p=a+1`. `p[1]`=`a[2]`=**3**, `*(a+3)`=`a[3]`=**4**, `p[-1]`=`a[0]`=**1**.

### 문제 10 · call-by-reference (swap)
```c
void swap(int *x, int *y){ int t=*x; *x=*y; *y=t; }
int a=1, b=2; swap(&a,&b);
printf("%d %d", a, b);
```
??? success "정답: `2 1`"
    - 주소를 넘겨 원본을 교환. 값으로 넘겼다면 `1 2` 그대로 — 이게 핵심 함정.

## 시험장 체크리스트
- [ ] `*(p++)` 와 `*(++p)` 구분
- [ ] `int/int` 는 소수를 버린다
- [ ] `->` 는 포인터, `.` 는 값 변수
- [ ] `static`/전역은 호출 사이 값 유지
- [ ] 비트연산은 2진수로 그려 계산
- [ ] 재귀는 종료 조건부터 손으로 펼친다

!!! warning "시험은 스스로"
    시험은 **AI 없이** 손으로 추적하는 힘을 본다. 평소 연습은 AI와, 시험은 스스로.
