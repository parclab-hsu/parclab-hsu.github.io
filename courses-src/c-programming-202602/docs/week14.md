## 강의 해설

14주차는 서로 다른 종류의 데이터를 하나의 의미 있는 묶음으로 만드는 방법을 배운다. 지금까지 배열은 같은 자료형을 여러 개 담는 데 적합했다. 하지만 실제 센서 패킷은 거리, 속도, 상태 문자열처럼 서로 다른 자료형이 함께 움직인다. 구조체는 이런 값들을 "하나의 사건" 또는 "하나의 메시지"로 묶어 주는 도구다.

구조체를 배우면 프로그램의 표현력이 크게 좋아진다. `distance_cm`, `speed_cmps`, `state`가 따로 흩어져 있으면 이 값들이 서로 관련 있다는 사실을 사람이 기억해야 한다. `SensorPacket`으로 묶으면 코드 자체가 "이 세 값은 함께 전송되는 센서 패킷"이라고 말해 준다. 좋은 구조체 설계는 좋은 변수 이름처럼 프로그램을 읽기 쉽게 만든다.

직렬화와 역직렬화는 구조체를 통신으로 연결하는 핵심 기술이다. 메모리 안의 구조체를 그대로 보내는 대신 `"S,42.0,25.0,RUN"` 같은 문자열로 바꾸면 시리얼, WiFi, 로그에서 쉽게 확인할 수 있다. 반대로 받은 문자열을 다시 구조체로 바꾸면 C 코드가 다루기 쉬운 형태가 된다. 이 과정은 ROS2 메시지를 이해하는 준비 단계이며, `geometry_msgs/Twist` 같은 표준 메시지도 결국 이름 붙은 필드들의 묶음이다.

## 3시간 강의 운영 포인트

- **0~25분**: 배열과 구조체의 차이를 비교한다. 같은 종류 여러 개는 배열, 서로 다른 값을 하나의 의미로 묶을 때는 구조체라는 기준을 세운다.
- **25~80분**: `typedef struct`, 멤버 접근 `.`, 구조체 포인터 `->`를 예제로 설명한다. `SensorPacket`을 만들고 필드별 의미를 말하게 한다.
- **80~130분**: 구조체 배열과 직렬화/역직렬화 실습을 진행한다. 문자열 `"S,42.0,25.0,RUN"`을 구조체로 바꾸고 다시 출력하게 한다.
- **130~180분**: 연결리스트 한 노드 잇기와 ROS2 메시지 연결을 맛보기로 다룬다. 기말 프로젝트에서는 "데이터 묶음 설계"가 코드 품질을 좌우한다는 점을 강조한다.

## 강의 본문 보강

### 개념을 더 풀어 설명하기
구조체는 서로 다른 자료형의 값을 하나의 의미 단위로 묶는 방법이다. 배열은 같은 자료형을 여러 개 담는 데 적합하지만, 실제 센서 패킷은 거리, 속도, 상태 문자열처럼 서로 다른 종류의 값을 함께 다룬다. 이런 값을 따로따로 들고 다니면 관계가 흐려지고 실수하기 쉽다.

`SensorPacket` 같은 구조체를 만들면 코드가 더 설명적으로 바뀐다. `p.distance_cm`, `p.speed_cmps`, `p.state`는 각각의 값이 센서 패킷 안에 속한다는 사실을 드러낸다. 좋은 구조체 설계는 데이터 사이의 관계를 코드에 남기는 일이다.

### 통신과 구조체
구조체는 메모리 안에서 다루기 좋은 형태이고, 문자열 패킷은 통신과 로그에서 확인하기 좋은 형태다. 그래서 직렬화는 구조체를 문자열로 바꾸는 과정이고, 역직렬화는 문자열을 다시 구조체로 바꾸는 과정이다. `"S,42.0,25.0,RUN"`을 구조체로 해석하는 과정은 ROS2 메시지를 이해하는 준비가 된다.

### 학생 활동
- 거리, 속도, 상태를 따로 변수로 관리하는 코드와 구조체로 묶은 코드를 비교한다.
- 구조체 배열을 만들어 여러 센서 패킷을 저장한다.
- 구조체 포인터로 멤버에 접근할 때 `.`와 `->` 차이를 확인한다.
- 문자열 패킷을 구조체로 파싱하고, 다시 문자열로 출력한다.

### 자주 막히는 지점
- 구조체 변수는 `p.member`, 구조체 포인터는 `p->member`를 사용한다.
- 문자열 멤버는 배열 크기를 충분히 잡아야 한다.
- 통신에서는 구조체 메모리를 그대로 보내기보다 사람이 읽을 수 있는 형식으로 바꾸는 것이 디버깅에 좋다.
- 연결리스트는 깊게 들어가기보다 "구조체 안에 다음 노드 주소가 들어갈 수 있다"는 감각만 잡는다.

## 1. 이론

### 1.1 구조체란
여러 자료형을 **하나의 의미 단위**로 묶는다.
```c
typedef struct {
    float distance_cm;
    float speed_cmps;
    char  state[16];
} SensorPacket;            // 센서 패킷 = 통신의 한 묶음

SensorPacket p = {42.0f, 25.0f, "RUN"};
printf("%.1f %s\n", p.distance_cm, p.state);   // 멤버 접근: .
```

### 1.2 구조체 포인터 → `->`
```c
SensorPacket *pp = &p;
printf("%s\n", pp->state);   // (*pp).state 와 같다
```

### 1.3 구조체 배열
```c
SensorPacket fleet[3];
fleet[0].distance_cm = 10.0f;
```

### 1.4 직렬화/역직렬화 (통신의 핵심)
구조체를 **문자열로 변환(직렬화)** 해 보내고, 받아서 **다시 구조체로(역직렬화)**.
```c
// 직렬화: 구조체 → "S,42.0,25.0,RUN"
snprintf(buf, n, "S,%.1f,%.1f,%s", p.distance_cm, p.speed_cmps, p.state);
// 역직렬화: 문자열 → 구조체
sscanf(line, "S,%f,%f,%15[^\r\n]", &p.distance_cm, &p.speed_cmps, p.state);
```
> 이 아이디어가 곧 15주차 ROS2 메시지(geometry_msgs/Pose, Twist)와 똑같다.

### 1.5 연결리스트 맛보기 (자료구조 가교)
구조체가 **자기 자신을 가리키는 포인터**를 가지면 노드를 줄줄이 이을 수 있다.
```c
typedef struct Node {
    int value;
    struct Node *next;   // 다음 노드 주소
} Node;
Node a = {1, NULL}, b = {2, NULL};
a.next = &b;             // a → b 연결
printf("%d %d\n", a.value, a.next->value);  // 1 2
```
> 2학년 「자료구조 및 알고리즘」의 출발점.

### 실습 14-4 · 아두이노 구조체 직렬화 (`code/arduino/15_struct_packet`)
`SensorPacket` 구조체를 채워 `"S,거리,속도,상태"` 한 줄로 직렬화한다.
이 송신 형식은 15주차 `mobility_node`/ROS2 브리지와 그대로 호환된다(구조체→ROS2 메시지의 씨앗).
```cpp
typedef struct { float distance_cm; float speed_cmps; char state[16]; } SensorPacket;
void serialize(const SensorPacket *p, char *out, int n) {
    snprintf(out, n, "S,%.1f,%.1f,%s", p->distance_cm, p->speed_cmps, p->state);
}
```
추가 하드웨어 없이 동작(가상 센서).

### 실습 14-5 · PC에서 먼저 검증하기

구조체 직렬화는 Arduino에 올리기 전에 PC C 코드로 먼저 확인할 수 있다.

```bash
cd courses-src/c-programming-202602/docs/code/c/examples
gcc -Wall -Wextra -std=c11 ex06_pose_struct.c -o ex06_pose_struct
./ex06_pose_struct
gcc -Wall -Wextra -std=c11 packet_parser.c -o packet_parser
./packet_parser
```

예상 출력:

```text
serialized: P,1.25,-0.40,90.0
parsed: x=1.25 y=-0.40 yaw=90.0
distance=42.0 cm
speed=25.0 cm/s
state=RUN
```

!!! tip "구조체와 ROS2 메시지"
    `SensorPacket`은 우리가 직접 만든 메시지 묶음이고, ROS2의 `geometry_msgs/Twist`도 같은 생각으로 만든 표준 메시지 묶음이다. 이름과 필드가 정해져 있을 뿐, 핵심은 **관련 데이터를 하나로 묶는다**는 점이다.

## 연습문제
1. 구조체 **포인터**로 멤버에 접근하는 연산자는?
2. `SensorPacket p;` 에서 `p.state` 와 `pp->state`(`pp=&p`)의 차이는?
3. 연결리스트에서 다음 노드를 가리키기 위해 구조체가 갖는 멤버는?

??? success "정답 및 해설"
    1. `->` (화살표).
    2. `.`는 **구조체 변수**의 멤버, `->`는 **구조체 포인터**의 멤버 접근. `pp->state == (*pp).state`.
    3. **자기 자신을 가리키는 포인터** 멤버(예: `struct Node *next;`).
