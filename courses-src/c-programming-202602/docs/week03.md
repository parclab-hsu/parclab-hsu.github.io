## 강의 해설

3주차부터 C가 다른 입문 언어보다 조금 엄격하게 느껴지기 시작한다. C에서는 변수를 만들 때 "어떤 종류의 값을 담을 것인지"를 먼저 정해야 한다. 같은 숫자처럼 보여도 `int`, `float`, `double`, `char`는 메모리 크기와 표현 방식이 다르다. 이 차이를 이해하지 못하면 나눗셈 결과가 예상과 다르거나, 센서값이 잘리거나, 입력이 제대로 저장되지 않는 문제가 생긴다.

모빌리티 예제로 생각하면 자료형 선택은 더 현실적이다. 바퀴 속도는 실수일 수 있고, 기어 상태는 문자 하나일 수 있으며, 장애물 감지 여부는 0 또는 1로 표현할 수 있다. 거리와 시간처럼 물리량을 다루는 값은 소수점이 필요하므로 `double`이 자연스럽고, 카운터나 반복 횟수는 `int`가 적합하다. 즉 자료형은 문법 문제가 아니라 "데이터의 성격을 설계하는 결정"이다.

`scanf_s`에서 `&`를 붙이는 이유는 12주차 포인터의 예고편이다. 입력 함수는 변수의 값을 직접 바꾸어야 하므로, 변수 자체가 아니라 변수의 주소를 알아야 한다. 지금은 `&speed`를 규칙처럼 익히고, 후반부 포인터에서 그 의미를 다시 해석한다. 캐스팅도 마찬가지다. `(double)a / b`는 단순한 괄호가 아니라 "이 계산을 실수 계산으로 보라"고 컴파일러에 알려 주는 명시적 지시다.

## 3시간 강의 운영 포인트

- **0~25분**: `int`, `double`, `char`를 실제 물리량과 연결한다. 속도, 거리, 상태 문자처럼 데이터의 성격이 자료형 선택을 결정한다는 점을 강조한다.
- **25~80분**: 변수 선언, 초기화, 자료형 크기, 범위를 설명하고 `sizeof` 실습으로 눈에 보이게 만든다.
- **80~135분**: `scanf_s` 입력 실습을 진행한다. `&`를 뺐을 때의 오류, 정수 나눗셈과 실수 나눗셈의 차이를 반드시 손으로 추적하게 한다.
- **135~180분**: 센서값 입력-계산-출력 미션을 수행하고, 학생별로 "왜 이 자료형을 골랐는지"를 한 문장으로 설명하게 한다.

## 강의 본문 보강

### 개념을 더 풀어 설명하기
변수는 이름 붙은 저장 공간이다. 하지만 C에서는 저장 공간을 만들 때 반드시 자료형을 함께 정해야 한다. 자료형은 단순한 문법 장식이 아니라 저장할 수 있는 값의 범위, 메모리 크기, 계산 방식, 출력 방식까지 결정한다. 같은 `10`이라도 `int`로 다룰 때와 `double`로 다룰 때 계산 결과가 달라질 수 있다.

모빌리티 예제로 보면 자료형 선택은 현실적인 설계 결정이다. 바퀴 회전 수는 정수 카운터일 수 있고, 주행 속도나 거리 센서값은 실수일 수 있다. 상태 문자는 `R`, `S`, `L`처럼 `char` 하나로 표현할 수 있다. 잘못된 자료형을 선택하면 센서값이 잘리거나 평균 계산이 틀리고, 제어 판단도 달라질 수 있다.

### 라이브 코딩 흐름
먼저 `sizeof`로 자료형 크기를 출력한다. 그 다음 `int a = 7, b = 2;`에서 `a / b`와 `(double)a / b`의 결과를 비교한다. 마지막에는 거리와 시간을 입력받아 속도를 계산하는 프로그램을 만든다. 학생에게 입력값 `100 3`을 넣게 하고, 정수 계산과 실수 계산의 차이를 직접 확인하게 한다.

```c
double distance_m;
double time_s;
scanf_s("%lf %lf", &distance_m, &time_s);
printf("speed = %.2f m/s\n", distance_m / time_s);
```

여기서 `&distance_m`은 "변수에 값을 넣을 주소를 알려 준다" 정도로 설명하고, 포인터의 자세한 의미는 12주차에 다시 다룬다고 예고한다.

### 학생 활동
- 속도, 배터리 잔량, 기어, 오류 코드에 적합한 자료형을 고르고 이유를 설명한다.
- `sizeof(int)`, `sizeof(double)`, `sizeof(char)` 결과를 표로 정리한다.
- 정수 나눗셈이 필요한 경우와 실수 나눗셈이 필요한 경우를 각각 예로 든다.
- `scanf_s`에서 `&`를 지운 뒤 어떤 오류가 생기는지 관찰한다.

### 자주 막히는 지점
- `%f`와 `%lf`는 `printf`와 `scanf_s`에서 쓰임이 다르게 느껴질 수 있으므로 예제로 구분한다.
- 정수끼리 나누면 결과도 정수로 잘릴 수 있다.
- 입력 형식과 실제 입력값의 개수가 맞지 않으면 프로그램이 예상대로 진행되지 않는다.
- 변수 이름은 의미 있게 지어야 이후 함수와 구조체로 확장할 때 읽기 쉽다.

## 1. 이론

### 1.1 변수와 메모리
- **변수**: 값을 담는 이름 붙은 메모리 공간. `int a = 10;` → 4바이트 공간에 10 저장.
- 변수는 **선언(자료형 지정) → 사용** 순서. 자료형이 "그릇의 크기·종류"를 정한다.

### 1.2 기본 자료형
| 자료형 | 크기(보통) | 범위/용도 | 형식지정자 |
|--------|-----------|-----------|------------|
| `int` | 4바이트 | 약 ±21억 정수 | `%d` |
| `float` | 4바이트 | 실수(정밀도 7자리) | `%f` |
| `double` | 8바이트 | 실수(정밀도 15자리) | `%lf` |
| `char` | 1바이트 | 문자/작은 정수 | `%c` |
| `unsigned int` | 4바이트 | 0 이상 정수 | `%u` |

> `sizeof(자료형)`으로 크기를 확인할 수 있다: `printf("%zu", sizeof(int));` → 4

### 1.3 상수
```c
const double PI = 3.14159;   // const: 변경 불가 변수
#define MAX_SPEED 120        // 전처리기 매크로 상수
```

### 1.4 형 변환(캐스팅)
- **암묵적**: 서로 다른 자료형 연산 시 자동 변환.
- **명시적**: `(double)a` 처럼 직접 지정. **정수 나눗셈 함정**을 피하는 핵심.
```c
int a = 7, b = 2;
printf("%d\n", a / b);          // 3  (정수 나눗셈, 소수점 버림)
printf("%.1f\n", (double)a / b);// 3.5 (캐스팅 필요)
```

### 1.5 입력: scanf_s
```c
int speed;
scanf_s("%d", &speed);   // &speed: 값을 저장할 변수의 '주소'
```
!!! warning "자주 하는 실수"
    - `scanf`에 **`&`(주소)** 를 빠뜨리면 안 된다(문자열 배열 제외).
    - `float`는 `%f`, `double` 입력은 `%lf`.
    - 부동소수는 오차가 있다(0.1을 정확히 못 담음).

## 2. 핵심 용어 정리
| 용어 | 설명 |
|------|------|
| 변수 | 값을 담는 이름 붙은 메모리 공간 |
| 자료형 | 값의 종류·크기(int/float/char…) |
| 리터럴 | 코드에 직접 쓴 값(`10`, `3.14`, `'A'`) |
| 상수 | 변하지 않는 값(`const`, `#define`) |
| 캐스팅 | 자료형을 바꾸는 것 `(double)a` |
| 오버플로 | 자료형 범위를 넘어 값이 깨지는 현상 |
| 부동소수점 | 실수를 근사 표현하는 방식(오차 존재) |

## 3. 실습

### 실습 3-1 · 입력→출력
거리(cm)·시간(s)을 입력받아 그대로 출력.
```c
double dist, t;
scanf_s("%lf %lf", &dist, &t);
printf("거리 %.1f cm, 시간 %.1f s\n", dist, t);
```

### 실습 3-2 · 자료형 크기 비교
```c
printf("int=%zu float=%zu double=%zu char=%zu\n",
       sizeof(int), sizeof(float), sizeof(double), sizeof(char));
```

### 실습 3-3 · 캐스팅 실험
정수 7/2와 실수 7/2를 모두 출력하고 차이를 주석으로 설명(연습 1-3).

## 연습문제
1. `int a=7,b=2; printf("%d", a/b);` 의 출력은?
2. `printf("%.1f", (double)7/2);` 의 출력은?
3. `scanf_s`로 정수를 받을 때 변수 앞에 반드시 붙여야 하는 기호는?
4. `double` 값을 입력받을 때 쓰는 형식 지정자는?

??? success "정답 및 해설"
    1. `3` — 정수끼리 나눗셈은 몫만 남는다(소수점 버림).
    2. `3.5` — 한쪽을 `(double)`로 캐스팅하면 실수 나눗셈.
    3. `&` (주소 연산자). 예: `scanf_s("%d", &n);`
    4. `%lf`
