# 📊 2026 트렌드 기반 교육과정 검토

> 한성대 미래모빌리티 **직무역량 분석** + **2026 기술 트렌드**로 본 강의 설계 근거.
> C 프로그래밍은 1학년 2학기 **SW 기초** 교과로, 이후 임베디드·ROS·차량통신으로 이어진다.

## 후속 교과 연결 (C가 첫 관문)
```
1-2 C프로그래밍 → 2-1 모빌리티 프로그래밍 실습 · 자료구조 · 논리회로/컴퓨터구조
   → 3-1 임베디드시스템·OS · 센서처리와 모터제어
   → 3-2 ECU 실습 · Physical AI & 로보틱스 시뮬레이션
   → 4-1 이동로봇과 ROS · 모빌리티 SW공학  → 4-2 모빌리티 통신(CAN·Ethernet)
```

## 직무 연결 지도 (교과 개념 → 채용 스택)
| 배우는 것 | 직결 직무 스택 |
|-----------|----------------|
| 구조체·직렬화 | 센서 패킷·메시지·CAN 프레임 |
| 포인터·메모리 | 임베디드·실시간·ECU 펌웨어 |
| 시리얼/WiFi 통신 | CAN·LIN·차량 Ethernet |
| C↔ROS2 연동 | ROS2·DDS·미들웨어 |
| Git·AI 활용 | 협업·생산성(신입 표준) |

> 신입 표준 진입 스택: Python + PyTorch + **Linux + Git + C/C++**

## 2026 기술 트렌드 요지
- **Physical AI**: 화면 속 AI → 물리세계 인지·판단·행동. 실시간 제어·펌웨어는 **C/C++**. *"AI는 두뇌, C는 신경·근육."*
- **ROS2**: 현행 LTS = **Jazzy**(Humble도 광범위). C 진입점 = rcl / **rclc(micro-ROS)**. UNO R4 micro-ROS **공식 지원**(경량).
- **Rust vs C**: 자동차/임베디드는 여전히 **C 지배**(MISRA C:2025, AUTOSAR, ISO 26262). Rust는 보완재.
- **TinyML/Edge AI**: MCU에서 ML 추론(TFLite Micro·Edge Impulse). 학습 모델을 **C 배열**로 내보내 호출.
- **SDV**: 소프트웨어 정의 차량. Classic(C)·Adaptive(C++) 공존. 딥임베디드 실시간 = C.
